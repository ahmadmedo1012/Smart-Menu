## 🔴 خطير (1)

### 1. تسجيل الدخول لا يعمل عبر النموذج مباشرة
**المسار**: `/login`
**التفاصيل**: عند ملء admin/admin123 والضغط على "تسجيل الدخول"، لا يُرسل الطلب إلى `/api/auth/login`. لا يظهر أي خطأ في الـ Network tab. لكن الـ fetch المباشر عبر الـ Console ينجح (200). السبب المحتمل: `csrfFetch` يفشل لأن الكوكي `csrf-token` قد لا يكون موجوداً على النطاق المباشر (مصمم لـ localhost).
**الأثر**: المستخدم لا يستطيع تسجيل الدخول عبر الواجهة (لكن الوكلاء الآخرين نجحوا — قد يكون خطأ متقطع).

## 🟡 رئيسي (5)

### 2. إحصائيات الصفحة الرئيسية عالقة على 0
**المسار**: `/` (landing)
**التفاصيل**: جميع الأرقام تظهر 0 (0+ مطعم، 0+ طلب يومي، 0% رضا). عداد التفعيل لا يعمل أو الأرقام hardcoded بـ 0 بدل القيم الحقيقية مثل 500+.

### 3. Manifest.json محجوب بواسطة CSP
**المسار**: `/manifest.json`
**التفاصيل**: `manifest-src` غير مضبوط في CSP، فـ default-srt 'self' يحجب إعادة التوجيه إلى vercel.com/sso-api. يمنع PWA installability.

### 4. حماية Vercel تعترض /admin قبل Middleware
**المسار**: `/admin`
**التفاصيل**: بسبب Vercel Deployment Protection، المستخدم غير الموثّق يُوجّه لصفحة Vercel، وليس لـ Next.js middleware. المشكلة في تكوين Vercel.

### 5. صفحة /login متاحة حتى بعد تسجيل الدخول
**المسار**: `/login`
**التفاصيل**: المستخدم المسجل دخوله يزور /login ويراها فارغة — لا إعادة توجيه. يحتاج فحص session عند تحميل الصفحة.

### 6. تناقض في عدد الطلبات بين Dashboard وصفحة الطلبات
**المسار**: `/owner/orders`
**التفاصيل**: الـ dashboard يظهر 27 طلباً، صفحة الطلبات تظهر 20. تصفية الحالة لا تعمل (الكل 20، لكن dashboard يظهر 25 جديد).
