## 3. تقييم التصميم

### 3.1 التسلسل الهرمي البصري (Visual Hierarchy)

**ما يعمل**:
- الهيرو (Hero) بارز: استخدام gradient وصورة خلفية مع تأثير شبكي (grid pattern)
- العناوين واضحة وكبيرة في قسم الهيرو
- أزرار CTA بيضاء على خلفية برتقالية — تباين جيد
- الهيدر ثابت في الأعلى بارتفاع 68px مع خلفية شفافة تتحول

**ما يحتاج تحسين**:
- أزرار CTA في الموبايل: صغيرة نسبياً (17px font, height غير محدد)
- تأثير الـ backdrop-filter / glassmorphism في الهيدر: شفاف جداً ويحتاج تحسين تباين readability
- Scroll-to-top button: مخفي افتراضيًا (opacity-0, translate-y-4) ويظهر عند السكرول — لكن بدايته خاطئة (pointer-events-none)
- الـ feature cards الستة كلها متطابقة تماماً في الحجم (154×456px) وعديمة الظل (shadow: none) — لا يوجد تمييز بصري للترتيب

### 3.2 التكرار البصري — قسم الميزات (FeaturesSection)

تم مسح 6 كروت مميزات مباشرة من DOM:

| الخاصية | القيمة | ملاحظة |
|---------|--------|--------|
| الخلفية | `lab(100% 0 0)` (أبيض) | متطابقة للكل |
| الظل | `none` | لا ظل — الكروت مسطحة |
| border-radius | `16px` | موحد |
| الارتفاع | `154px` | متطابق |
| العرض | `456px` | متطابق |
| المحتوى | أيقونة SVG، عنوان، وصف | نفس الهيكل |

**مستوى التكرار**: 6/6 كروت متطابقة تماماً في الحجم والشكل والظل. لا يوجد تمييز (highlight) لأي ميزة رئيسية. هذا يجعل القسم يبدو وكأنه قائمة عادية وليس hierarchy مقروء.

> **التقييم**: درجة التكرار البصري 0% تمايز — كل الكروت نسخة واحدة. يحتاج إضافة بطاقة مميزة (featured card) أو تفاوت في الأحجام أو إضافة ظلال متدرجة حسب الأهمية.

### 3.3 جودة الصور — المنيو الرقمي

**عدد عناصر `<img>` في صفحة المنيو**: 0 (صفر)
**عناصر الخلفية (background-image)**: 0

صفحة `/menu/al-waha-cafe` لا تحتوي على أي صور حقيقية. العناصر البصرية الوحيدة:
- أيقونات SVG (Lottie animation في hero — عبارة عن SVG مضمنة inline base64)
- أشكال blob متحركة CSS
- Grid pattern SVG

**المشكلة**: قائمة المطعم الرقمية تخلو تماماً من صور الأطباق والمشروبات. العناصر تظهر كنص فقط (اسم + سعر). هذا يؤثر سلباً على جاذبية المنيو.

> **التقييم**: درجة 0/10 للصور. لا توجد صور طعام حقيقية.

### 3.4 الوضع الداكن / الفاتح

**الحالي**: الوضع الفاتح نشط (`data-theme` غير موجود، class `dark` غير موجود)
**المكتشف**:
- يوجد زر تبديل في الهيدر مع أيقونة شمس (نشط حالياً) — `aria-label="الوضع الليلي"`
- الثيم يُحمّل عبر script inline: يقرأ `localStorage.getItem("theme")` ويدعم light/dark
- متغيرات CSS للداكمود موجودة (تُشاهد في خلفية radial-gradient: `dark:bg-[radial-gradient(...)]`)
- `colorScheme` مضبوط في CSS (يُرى في inline script)

**التباين المقاس في الوضع الفاتح**:
- `--primary` (برتقالي غامق) على `--primary-foreground` (أبيض): نسبة تباين عالية جدا
- `--muted-foreground` (رمادي 44%) على `--muted` (93%): نسبة تباين ≈ 6.5:1 — جيد
- `--card-foreground` (أسود) على `--card` (أبيض): تباين ممتاز

> **التقييم**: الزر موجود، الكود الخلفي يدعم التبديل. لم يتم اختبار الدارك مود فعلياً لأنه يحتاج تفعيل يدوي، لكن البنية التحتية سليمة.

### 3.5 جودة الحركة (Motion)

**المكتشف**:
- عناصر متحركة CSS: 30 عنصر تحوي `[style*="transform"]` (عدد المقاس)
- تأثير blob المتحرك: 3 عناصر في hero مع `animation-duration: 22s, 26s, 19s`
- Header له `transition-all duration-300`
- زر العودة للأعلى: `transition-all duration-300` مع opacity وtransform
- Lottie animation: SVG مضمّن في hero (dotlottie-container مع صورة base64)
- `animate-float` class: تأثير عائم على أيقونة السهم

**لم يُعثر على**:
- Framer Motion لم يُكتشف (لا في scripts ولا في `window`)
- GSAP غير موجود
- AOS غير موجود

**التقييم**: الحركات تعتمد على CSS Animations/Transitions النقية و Lottie SVG. لا يوجد مكتبة JS للحركات (Framer Motion) — مما يعني عدم وجود enter/exit animations معقدة. الانتقالات سلسة لكونها CSS-only. الحركات بسيطة والموقع لا يعاني من stutter observable.

### 3.6 RTL (من اليمين لليسار)

**المقاييس**:
- `html dir="rtl"` ✓
- `html lang="ar"` ✓
- الهيدر: العناصر تطبيق RTL بشكل صحيح (button مع `start-4` بدل `left-4`)
- استخدام `inset-x-0` (يمين ويسار معاً) و `end-8` (right في RTL) — صحيح
- المحتوى العربي في الصفحة مكتمل

> **التقييم**: دعم RTL ممتاز. كل شيء يبدو صحيحاً من ناحية الاتجاه.
